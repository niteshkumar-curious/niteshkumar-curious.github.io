ESP32 OTA Firmware Update System

This project demonstrates an IoT-based remote device management system using ESP32, where firmware updates can be performed over-the-air (OTA) without physical access. It enables efficient, scalable, and secure firmware deployment for embedded systems.

📌 Problem Statement

Traditional embedded systems require physical connection (USB/UART) for firmware updates. This is inefficient for deployed or remote devices. The goal is to build a system that allows remote firmware updates securely using the internet.

⚙️ System Architecture

  • ESP32 (ESP-IDF) for firmware execution
  • MQTT for communication and remote commands
  • HTTPS for secure firmware download
  • GitHub for firmware hosting
  • Web dashboard (Netlify/Render) for control and monitoring

🚀 Features

  • Over-the-Air firmware updates
  • Secure HTTPS firmware download
  • MQTT-based remote control
  • Real-time OTA progress tracking
  • FreeRTOS-based multitasking
  • Cloud-based firmware deployment

🧠 Working

The ESP32 connects to WiFi and subscribes to MQTT topics. When an OTA command is received, the device downloads firmware from a GitHub-hosted URL using HTTPS. The update runs in real-time, and progress is sent back via MQTT. After completion, the device restarts with the updated firmware.

📊 OTA Flow

  1. Device connects to WiFi and MQTT broker
  2. User sends OTA command
  3. ESP32 downloads firmware via HTTPS
  4. Progress (0–100%) is published
  5. Device restarts with new firmware

💻 OTA Code Snippet


esp_http_client_config_t http_config = {
    .url = OTA_FIRMWARE_URL,
    .crt_bundle_attach = esp_crt_bundle_attach,
};

esp_https_ota_config_t ota_config = {
    .http_config = &http_config,
};

esp_https_ota(&ota_config);

The firmware binary is hosted on GitHub and accessed by the ESP32 during OTA updates. This enables easy version control and remote deployment.
